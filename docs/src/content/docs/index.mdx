<div class="dux-section">
  <div class="dux-showcase not-content">
    <div>
      <span class="eyebrow">Design Tradeoffs</span>
      <h2>Keep ORM boundaries visible in code</h2>
      <p>Oro does not hide complexity behind generators or pack schema settings into tag strings. Models, tables, and raw SQL are separate entries: use models for business code, tables for dynamic flows, and Raw for complex queries.</p>
      <ul>
        <li>Schema lives in `Define`, not in overloaded tags.</li>
        <li>Writes are explicit, so zero values are not guessed.</li>
        <li>Relations live in methods, reducing import-cycle pressure.</li>
      </ul>
    </div>
    <ShowcaseCode code={`func (Product) Define(s *oro.SchemaBuilder) {
    s.Table("products")
    s.Field("Code").String().Unique()
    s.Field("Price").Uint().Default(0)
}

db.Use[Product]()      // *Product / []Product
db.Table("products")  // oro.Map
db.Raw("select ...")  // Map or MapTo[T]()`} />
  </div>
</div>
