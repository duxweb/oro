---
title: 结构同步
description: 不写迁移文件，直接把注册模型同步为数据表。
---

Oro 使用模型定义作为结构来源。

```go
if err := db.Register(Product{}, User{}, ArticleTag{}); err != nil {
    return err
}
if err := db.Sync(ctx); err != nil {
    return err
}
```

## 自动执行的安全变更

- 创建缺失表；
- 添加缺失字段；
- 创建索引和唯一索引；
- 创建驱动支持的全文索引；
- 维护内部结构快照用于后续 diff。

## 拒绝的危险变更

- 删除字段；
- 缩窄字段类型；
- 收紧 nullable 约束；
- 不明确的类型变更；
- 无法判断的字段重命名。

原则很简单：自动同步只做安全的向前变更。破坏性数据操作应该由显式应用代码或人工数据库操作完成。

## 字段类型变更

兼容的扩大类型可以由方言在安全时处理。不安全变更会被拒绝，因为不同数据库的类型转换和数据丢失行为并不一致。

推荐流程：新增字段、显式回填、部署读取新字段的代码，然后在安全窗口手动移除旧字段。

## 外键

数据库级外键默认关闭。关联首先是 ORM 元数据和查询行为。需要数据库级约束时，再通过字段或同步配置显式开启。

这样能让 SQLite、MySQL、PostgreSQL、分片表和模块化项目都保持同步策略稳定。
